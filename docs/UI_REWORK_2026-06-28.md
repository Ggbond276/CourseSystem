## 一、重构动机

之前的前端是「按模块拆页面」风格：教师 5 个页面、学生 4 个页面，每个操作都要跳一次路由。
带来三个问题：

1. **菜单喧宾夺主** — 侧边栏罗列 8-10 个菜单项，用户一眼看不清「我现在在哪 / 我要去哪」。
2. **操作流被打断** — 创建课程要跳 `CreateCourse.vue`、发布作业要跳 `CreateHomework.vue`，路由一跳浏览器地址栏就变，刷新页面容易丢状态。
3. **角色心智不一致** — 教师/学生看到的页面骨架完全不同，互相切换时需要重新理解。

---

## 二、重构后的设计哲学：「用户旅程驱动 UI」

每个页面只承担一个**完整的用户旅程**，中途不跳路由：

| 旅程 | 触发位置 | 完成的操作 |
|------|----------|-----------|
| **工作台** | 登录后默认页 | 看今日待办 + 快捷入口 |
| **课程大厅** | 侧边栏「我的课程」 | 创建 / 加入 / 进入课程 |
| **课程详情** | 课程卡片点击 | 课程信息 + 作业大厅 + 课内操作 |
| **作业详情** | 作业卡片点击 | 查看作业 + 提交作业 + 讨论区 |
| **批阅大厅** | 教师课程详情「作业大厅」里点作业 | 列表筛选 + 进入批改 |
| **批改作业** | 批阅大厅点击某条提交 | 打分 + 写评语 |

弹窗（Dialog / Drawer）承担所有「中途操作」，主体页面保持稳定不变。

## 五、各组员的板块归属（保持不变）

虽然前端文件路径变了，但**分工表不变**，原因：后端 Service/Controller 接口**完全没动**，前端页面的内部逻辑也只是重新组织，旧代码逻辑被吸收进了新文件。

| 组员 | 负责的前端文件（新版） | 负责的后端文件（不变） |
|------|----------------------|----------------------|
| **组员 A** | `views/course/MyCourses.vue`、`views/course/CourseDetail.vue`、`api/course.js` | `CourseTeacherController`、`CourseStudentController`、`ICourseService`、`IStudentCourseService` 及对应 Mapper/POJO/DTO |
| **组员 B** | `views/teacher/HomeworkSubmissions.vue`、`views/teacher/GradeDetail.vue`、`views/student/StudentHomeworkDetail.vue`、`api/homework.js`、`api/comment.js` | `HomeworkTeacherController`、`HomeworkStudentController`、`HomeworkCommentController` 及对应 Service/Mapper/POJO/DTO |
| **组员 C** | `views/login/LoginIndex.vue`、`store/user.js`、`api/auth.js` | `AuthController`、`IAuthService`、`User`、`UserCourse`、`UserMapper`、`UserCourseMapper` |

> ⚠️ 注意：`MyCourses.vue` 和 `CourseDetail.vue` 由组员 A 拥有，但被教师/学生**共用**，里面用 `<component>` + `role` 字段做分支渲染。
> 组员 B 想给 `CourseDetail.vue` 加作业相关 UI，**不要**直接改文件，而是提 PR 给组员 A，或在 `CourseDetail.vue` 内预留的 `<slot name="b1-content" />` 插槽里实现。
